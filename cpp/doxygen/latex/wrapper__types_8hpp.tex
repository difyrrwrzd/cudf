\subsection{Detailed Description}
Wrapper structs for for the non-\/fundamental gdf\+\_\+dtype types. 

These structs simply wrap a single member variable of a fundamental type called \char`\"{}value\char`\"{}.

These wrapper structures are used in conjunction with the type\+\_\+dispatcher to emulate \char`\"{}strong typedefs\char`\"{}, i.\+e., provide opaque types that allow template specialization. A normal C++ typedef is simply an alias and does not allow for specializing a template or overloading a function.

The purpose of these \char`\"{}strong typedefs\char`\"{} is to provide a one-\/to-\/one mapping between gdf\+\_\+dtype enum values and concrete C++ types and allow distinguishing columns with different gdf\+\_\+dtype types, but have the same underlying type. For example, the underlying type of both G\+D\+F\+\_\+\+D\+A\+T\+E32 and G\+D\+F\+\_\+\+I\+N\+T32 is int32\+\_\+t. However, if one wished to specialize a functor invoked with the type\+\_\+dispatcher to handle G\+D\+F\+\_\+\+D\+A\+T\+E32 different from G\+D\+F\+\_\+\+I\+N\+T32, that would not be possible with aliases.

The standard arithmetic operators are defined for these wrapper structs such that they can be used as if they were fundamental arithmetic types.

In general, interacting with the wrapper structs should be done via the defined operators. However, if one needs to directly access the underlying value, the \char`\"{}unwrap\char`\"{} function may be used. Calling {\ttfamily unwrap} on an instance of a wrapper struct will return a reference to the underlying value. Calling {\ttfamily unwrap} on an instance of a fundamental type will return a reference to that instance (effectively a no-\/op).
